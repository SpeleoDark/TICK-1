# TICK-1: Мониторинг CMS на стеке TICK

## Обзор

Для обеспечения наблюдаемости CMS WordPress был развёрнут стек мониторинга **TICK** (Telegraf, InfluxDB, Chronograf, Kapacitor).  
Все компоненты установлены на одной виртуальной машине Ubuntu 22.04.  
Мониторинг охватывает как системные ресурсы хоста, так и ключевые компоненты CMS: **nginx**, **php-fpm** и **MySQL**.

---

## Установленные компоненты

### Компоненты CMS
| Компонент | Роль |
|-----------|------|
| **nginx** | Веб-сервер, обрабатывающий входящие запросы, предоставляет статус-страницу stub_status для сбора метрик |
| **php-fpm** | Менеджер процессов PHP, обслуживает динамический контент CMS; экспортирует статус-страницу с информацией о процессах и медленных запросах |
| **MySQL** | Реляционная база данных, хранящая содержимое и настройки WordPress; предоставляет внутреннюю статистику через команды `SHOW STATUS` и др. |
| **WordPress** | Сама CMS, работающая на связке nginx + php-fpm + MySQL |

### Компоненты стека TICK
| Компонент | Роль |
|-----------|------|
| **Telegraf** | Агент сбора метрик. Собирает данные с хоста (CPU, память, диск, сеть), а также специфичные метрики с nginx, php-fpm, MySQL и отслеживает состояние процессов |
| **InfluxDB** | База данных временных рядов. Хранит все метрики, отправляемые Telegraf |
| **Chronograf** | Веб-интерфейс для визуализации и администрирования стека TICK. Используется для построения дашбордов и управления алертами |
| **Kapacitor** | Обработчик потоковых данных. На основе поступающих метрик выполняет скрипты TICKscript и генерирует алерты при нарушении порогов |

---

## Что мониторится

### Системные метрики (сбор через Telegraf)
- Загрузка CPU, использование памяти, заполнение диска
- Сетевой трафик
- Базовая информация о системе (нагрузка, uptime)

### Метрики nginx
- Количество принятых соединений, обработанных запросов
- Активные соединения (чтение, запись, ожидание)
- Количество ответов с кодом 2xx, 3xx, 4xx, 5xx

### Метрики php-fpm
- Количество активных и idle процессов
- Общее число обработанных запросов
- Число медленных запросов (превысивших `request_slowlog_timeout`)
- Пиковое использование памяти и загрузка очереди

### Метрики MySQL
- Количество выполняемых запросов (questions)
- Активные соединения (threads_connected)
- Статистика по типам команд (SELECT, INSERT и т.д.)

### Мониторинг процессов
- Отслеживается наличие процессов `nginx`, `php-fpm`, `mysqld`
- При падении любого из них генерируется алерт

## Сводный дашборд

В **Chronograf** построен дашборд, содержащий наиболее важные графики:

- **CPU Usage** – загрузка процессора
- **Memory Usage** – процент использованной памяти
- **Disk Usage** – занятое место на корневом разделе
- **Network Traffic** – входящий/исходящий трафик
- **Nginx Requests/s** – интенсивность запросов (RPS)
- **Nginx 5xx Errors** – ошибки серверной стороны
- **PHP-FPM Active Processes** – текущая нагрузка на пул процессов
- **PHP-FPM Slow Requests** – динамика медленных запросов
- **MySQL Queries** – количество запросов к БД
- **MySQL Connections** – количество подключений к MySQL
- **Processes Status** – индикатор работоспособности ключевых процессов

Все графики расположены на одном экране для быстрой оценки общего состояния системы.

## Правила алертинга

В **Kapacitor** загружены и активированы следующие правила (TICKscripts):

| Алерт | Условие срабатывания |
|-------|----------------------|
| **CPU High Usage** | Средняя загрузка CPU > 90% в течение 5 минут |
| **Memory High Usage** | Использование памяти > 90% в течение 5 минут |
| **Disk Space Low** | Заполнение корневого раздела > 85% |
| **Service Down** | Отсутствие процесса `nginx`, `php-fpm` или `mysqld` |
| **Nginx 5xx Errors** | Более 5 ошибок 5xx в секунду (среднее за минуту) |

Уведомления записываются в лог Kapacitor; в Chronograf можно просмотреть текущие срабатывания.
